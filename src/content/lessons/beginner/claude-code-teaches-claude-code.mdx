# Claude Code teaches Claude Code

## The shortest, longest-lived habit

This is the shortest lesson in the course, and the one you will use after everything else here has gone slightly out of date.

Here is the idea: **when you want to know how to do something in Claude Code, ask Claude Code.** It has its own documentation available to it, and it fetches the current version to answer, so you can just ask:

```prompt
how do I make you always run prettier after every edit?
```

Claude explains hooks, because that is what your question is really about, and it does so from today's docs, not a half-remembered version. Try any of these:

```prompt
can you create pull requests?
```

```prompt
how does permission mode "auto" differ from "accept edits"?
```

```prompt
what skills do I have available right now?
```

```prompt
how do I connect a database with MCP?
```

No special syntax, no command to memorize. You ask a question the way you would ask a senior teammate who happens to have read the entire manual this morning.

## Why the answer stays current

This matters more than it looks, and it is the reason to build the habit now.

Claude Code ships new features constantly. Any course, including this one, is a snapshot: it was accurate when written and drifts a little with every release after. You already know from <LessonLink id="B1.2" /> that the version you are running may be newer than the version this platform was built against.

The self-documentation habit is immune to that drift. **Claude always has access to the latest Claude Code documentation, regardless of which version you are running or when this lesson was written.** When the platform lags a release behind, asking Claude directly does not. That is why every feature lesson here carries an **Official docs** footer: this platform teaches the concepts, and the live docs, reachable through Claude or through that footer, carry the current specifics.

Treat asking Claude as your active freshness tool, and the product's changelog or What's new feed as the passive one: pull the answer you need now, and skim the feed occasionally to learn what became possible since.

## Try It

**Environment:** any Claude Code session. This costs nothing and touches no files.

Ask Claude two questions about itself:

1. One you already know the answer to from this level, for example *"what's the difference between `/clear` and `/compact`?"* Check its answer against <LessonLink id="B2.3" />'s command belt. It should line up, and where it adds detail, that detail is current.
2. One this course has **not** taught you, something you are genuinely curious about: *"how do I run you on a schedule?"* or *"what's an agent team?"* Read the answer.

**Success criteria:** the second answer taught you something the course did not, and you did not need this platform to get it. That is what you gain: you now have a way to keep learning Claude Code that does not depend on anyone updating a lesson.
